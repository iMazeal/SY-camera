# 架构设计与开发指南

## 软件架构

### 分层设计

```
┌──────────────────────────────────────────┐
│               UI 层 (PyQt5)               │
│  MainWindow / Viewfinder / ControlsPanel  │
├──────────────────────────────────────────┤
│           相机控制层 (Python)              │
│  CameraManager / CaptureService          │
├──────────────────────────────────────────┤
│        硬件抽象层 (Picamera2)              │
│  Picamera2 / libcamera / OpenGL ES       │
└──────────────────────────────────────────┘
```

### 模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| 应用入口 | `src/main.py` | 初始化 QApplication、Picamera2、启动主窗口 |
| 相机管理 | `src/camera_manager.py` | 封装 Picamera2 配置、预览启动、拍照、录像 |
| 主窗口 | `src/ui/main_window.py` | 整体布局管理，整合预览和控制面板 |
| 取景器 | `src/ui/viewfinder.py` | QGlPicamera2 封装，处理相机预览显示 |
| 控制面板 | `src/ui/controls.py` | 拍照按钮、模式切换、参数设置等控件 |
| 图片工具 | `src/utils/image_utils.py` | 图像处理、文件管理、EXIF 处理 |

### 数据流

```
用户触摸拍照按钮
    ↓
ControlsPanel 发送 Qt 信号 (capture_requested)
    ↓
CameraManager 创建 still configuration
    ↓
Picamera2.switch_mode_and_capture_file()
    ↓
完成后通过信号/槽通知 UI (done_signal)
    ↓
MainWindow 更新状态、刷新图库预览
```

## 开发流程

### 分支策略

```
main        — 稳定版本（对应 RPi 部署）
dev         — 日常开发分支
feature/*   — 功能分支（从 dev 切出）
fix/*       — 修复分支
```

### 开发步骤

1. 从 `dev` 切出功能分支 `feature/<name>`
2. 在开发机上编写代码、测试
3. 提交 PR 到 `dev`，代码审查
4. 合并到 `dev`，集成测试
5. 发布时合入 `main`，部署到 RPi

### 代码规范

- 遵循 PEP 8（使用 flake8 / pre-commit 检查）
- 类型注解：所有函数参数和返回值标注类型
- 文档字符串：模块和类级别的 docstring，函数级别的短注释
- 命名：
  - 类名：`PascalCase`
  - 函数/变量：`snake_case`
  - 常量：`UPPER_CASE`
- Qt 相关：
  - 信号名：`snake_case` + `_signal` 后缀
  - 槽函数：`on_` + `对象名_事件名` 模式

## 已知问题与记录

### OpenGL ES 硬件加速启用失败（RPi 3B+）

**现象：** QGlPicamera2 初始化时 `eglCreateWindowSurface` 返回 `EGL_BAD_ALLOC`，自动降级到软件预览模式。

**影响：** 软件预览模式通过 `post_callback` 获取帧数据，CPU 占用较高，但功能完全正常。

**排查记录：**
- `dtoverlay=vc4-kms-v3d` 已配置并重启
- `gpu_mem` 未调整（默认 64MB）
- `QT_QPA_PLATFORM=eglfs` 无效
- `glxinfo` 不可用（无 X11）

**后续尝试方向：**
1. 增大 GPU 内存：`gpu_mem=128` 到 `/boot/config.txt`
2. 检查 `/boot/config.txt` 中 `dtoverlay=vc4-kms-v3d` 是否被其他配置覆盖
3. 尝试禁用 `dtoverlay=vc4-fkms-v3d`（如有）
4. 确认 DSI 屏幕的 EGL/GLES 驱动是否正确加载：`vc4gencmd`
5. 升级 `python3-pyqt5` 和 `python3-picamera2` 到最新版本

## Picamera2 关键 API 参考

### 初始化与预览

```python
from picamera2 import Picamera2, Preview

picam2 = Picamera2()
# 推荐：创建预览配置，自动选择最优摄像头模式
config = picam2.create_preview_configuration(
    main={"size": (800, 480)},          # 匹配 DSI 屏幕分辨率
    display="main"
)
picam2.configure(config)
# QTGL 预览（硬件加速，推荐）
picam2.start_preview(Preview.QTGL)
picam2.start()
```

### 拍照（异步方式，推荐）

```python
# 在信号连接之后调用
cfg = picam2.create_still_configuration()
picam2.switch_mode_and_capture_file(cfg, "photo.jpg",
                                    signal_function=qpicamera2.signal_done)

# 完成后由 done_signal 通知
def capture_done(job):
    picam2.wait(job)
    # 恢复预览模式
    picam2.switch_mode_and_capture_file(None, None)
```

### QGlPicamera2 嵌入（关键模式）

```python
from picamera2.previews.qt import QGlPicamera2

# QGlPicamera2 是一个 QWidget，可以嵌入到任意布局
qpicamera2 = QGlPicamera2(picam2, width=800, height=480, keep_ar=True)
# 信号连接
qpicamera2.done_signal.connect(capture_done)

# 放入布局中
layout.addWidget(qpicamera2)
```
