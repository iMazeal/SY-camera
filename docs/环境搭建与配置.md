# 环境搭建与配置

## Python 版本选择：3.13

| 因素 | 说明 |
|------|------|
| RPi 3B+ 系统默认 | 树莓派 OS 64-bit 原生搭载 Python 3.13 |
| Picamera2 最低要求 | >= 3.9（setup.py），3.13 完全兼容 |
| 开发机兼容 | uv 支持 `uv python install 3.13` 安装 |

**统一使用 Python 3.13**，开发和部署环境版本完全一致，避免版本行为差异。

## 开发环境（x86_64）

### 前置条件

- Ubuntu 24.04（当前开发机系统）
- uv 0.11.6（当前已安装）

### 初始化项目

```bash
cd /home/mazeal/camera

# 安装 Python 3.13（如果尚未安装）
uv python install 3.13

# 使用 uv 初始化 Python 项目
uv init --python 3.13

# 创建虚拟环境（指定 Python 3.13）
uv venv --python 3.13

# 激活虚拟环境
source .venv/bin/activate

# 验证 Python 版本
python --version  # 应输出 Python 3.13.x

# 安装核心依赖
uv pip install numpy pillow

# 安装 Picamera2（开发环境仅用于 API 模拟/类型检查）
uv pip install picamera2

# 安装 GUI 依赖
uv pip install PyQt5 PyOpenGL
```

### 验证安装

```bash
python -c "from PyQt5.QtWidgets import QApplication; print('PyQt5 OK')"
python -c "from OpenGL import GL; print('PyOpenGL OK')"
python -c "from picamera2 import Picamera2; print('Picamera2 OK')"
```

### 配置与工具

| 工具 | 用途 | 安装方式 |
|------|------|----------|
| pre-commit | 代码风格检查 | `uv pip install pre-commit && pre-commit install` |
| pylint | Python 静态分析 | `uv pip install pylint` |
| pyright | 类型检查 | `uv pip install pyright` |

## 树莓派 3B+ 部署环境

### 系统准备

Raspberry Pi OS（64-bit）已预装 Python 3.13。

### 安装 Picamera2

**官方推荐通过 apt 安装**（确保 libcamera 底层库版本匹配）：

```bash
# 完整安装（含 GUI 依赖）
sudo apt update
sudo apt install python3-picamera2

# 验证安装
python3 -c "from picamera2 import Picamera2; picam2 = Picamera2(); print('Picamera2 OK')"
```

### 安装项目 Python 依赖

```bash
cd /home/mazeal/camera

# 安装 uv（如未安装）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 创建虚拟环境（使用系统 Python 3.13）
uv venv --python 3.13

# 激活
source .venv/bin/activate

# 安装依赖（Picamera2 已在系统级安装，不需要 pip 安装）
uv pip install numpy pillow PyQt5 PyOpenGL
```

### DSI 触摸屏配置

Raspberry Pi OS 通常自动识别 DSI 触摸屏。如需手动配置：

```bash
# 检查 /boot/config.txt 确保以下行存在
dtoverlay=vc4-kms-v3d

# 触摸屏旋转（如需）
# 在 /boot/config.txt 中添加
dtoverlay=vc4-kms-v3d,rotate=90
```

### 设置开机自启动（二选一）

#### 方式 1：桌面环境 autostart（推荐）

应用自带 `camera.desktop` 文件，复制到 autostart 目录即可：

```bash
mkdir -p ~/.config/autostart
cp ~/SY-camera/camera.desktop ~/.config/autostart/
```

树莓派启动进入桌面后会自动启动相机应用。如需停止：

```bash
rm ~/.config/autostart/camera.desktop
```

#### 方式 2：systemd 服务（无桌面环境）

```bash
# 创建 systemd 服务文件
sudo nano /etc/systemd/system/camera-app.service
```

```ini
[Unit]
Description=Raspberry Pi Camera App
After=graphical.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/SY-camera
Environment=DISPLAY=:0
ExecStart=/home/pi/SY-camera/start.sh
Restart=on-failure

[Install]
WantedBy=graphical.target
```

```bash
sudo systemctl enable camera-app.service
sudo systemctl start camera-app.service
```

## 跨环境注意事项

| 组件 | 开发机 (x86_64) | RPi 3B+ (ARM) |
|------|-----------------|----------------|
| Python | **3.13** (uv 管理) | **3.13** (系统自带) |
| Picamera2 | pip 安装（仅语法验证） | apt 安装（驱动摄像头） |
| PyQt5 | pip 安装 | apt 安装或 pip 安装 |
| 摄像头硬件 | 无 | 通过 libcamera 驱动 |
| 预览 | 软件模拟 | OpenGL ES 硬件加速 |

**编码注意事项：**
- 摄像头初始化应包含 try/except，开发模式下使用 mock 或无摄像头模式
- 所有 Picamera2 硬件相关代码仅在 `__name__ == '__main__'` 时执行
- GUI 层与相机控制层分离，便于开发环境独立测试 UI
