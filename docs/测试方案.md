# 测试方案

## 测试策略

由于 Picamera2 需要树莓派硬件（libcamera + 摄像头），测试分三个层次：

| 层级 | 范围 | 执行环境 | 工具 |
|------|------|----------|------|
| 单元测试 | 图像处理、文件操作、UI 逻辑 | 开发机 (x86_64) | pytest |
| 集成测试 | 相机控制层（mock 硬件） | 开发机 | pytest + unittest.mock |
| 硬件测试 | 完整拍照流程 | RPi 3B+ | 手动 + 脚本 |

## 单元测试

### 测试框架

使用 `pytest`：

```bash
uv pip install pytest pytest-qt
```

### 测试内容

| 模块 | 测试内容 | 是否可 mock |
|------|----------|------------|
| `image_utils.py` | 图像缩放/裁剪/旋转/EXIF 写入 | 是 |
| `controls.py` | 按钮状态、配置变更回调 | 是 |
| `camera_manager.py` | 配置生成、模式切换逻辑 | 是 |
| `main_window.py` | 窗口布局、信号连接 | 是 |

### 运行单元测试

```bash
# 安装测试依赖
uv pip install pytest pytest-qt

# 运行所有测试
uv run pytest tests/

# 指定测试文件
uv run pytest tests/test_image_utils.py -v

# 带覆盖率
uv run pytest tests/ --cov=src --cov-report=term
```

## 集成测试（开发机）

### Picamera2 Mock

Picamera2 在 x86_64 上无法访问摄像头硬件，需使用 mock：

```python
# tests/conftest.py
from unittest.mock import MagicMock, patch
import pytest

@pytest.fixture
def mock_picamera2():
    """模拟 Picamera2 实例，避免硬件依赖。"""
    with patch('picamera2.Picamera2') as MockPicamera2:
        mock_instance = MockPicamera2.return_value
        # 模拟常用方法
        mock_instance.create_preview_configuration.return_value = {
            "use_case": "preview",
            "transform": None,
            "queue": True,
            "main": {"size": (800, 480), "format": "RGB888"},
            "lores": None,
            "raw": None,
            "display": "main",
            "buffer_count": 2
        }
        mock_instance.create_still_configuration.return_value = {
            "use_case": "still",
            # ... same pattern
        }
        mock_instance.camera_config = None
        mock_instance.display_stream_name = "main"
        mock_instance.stream_map = {}
        mock_instance.verbose_console = False
        yield mock_instance
```

### 测试 CameraManager

```python
# tests/test_camera_manager.py
def test_create_preview_config(mock_picamera2):
    manager = CameraManager(mock_picamera2)
    config = manager.get_preview_config()
    assert config is not None
    assert config["main"]["size"] == (800, 480)
```

### 注意点

- Mock 无法测试 OpenGL 渲染路径（QGlPicamera2 需要 GPU）
- UI 测试可使用 `pytest-qt` 的 `qtbot` fixture
- 信号/槽连接测试不需要硬件

## 硬件测试（RPi 3B+）

### 测试清单

| 测试项 | 预期结果 | 验证方式 |
|--------|----------|----------|
| Picamera2 初始化 | 无异常 | `python3 -c "from picamera2 import Picamera2; Picamera2()"` |
| 摄像头预览 | 15fps+ 流畅显示 | 目测 |
| 触摸拍照 | 点击按钮后 200ms 内完成 | `time` 计时 |
| 图片保存 | JPEG 文件正确写入 SD 卡 | `file photo.jpg` |
| DSI 屏幕适配 | 全屏 800x480 无黑边 | 目测 |
| 连续拍照 | 100 次拍照无内存泄漏 | `free -h` 观察内存 |

### 硬件测试脚本

```bash
# 在 RPi 上快速验证相机
python3 << 'EOF'
from picamera2 import Picamera2
import time

picam2 = Picamera2()
config = picam2.create_preview_configuration(main={"size": (800, 480)})
picam2.configure(config)
picam2.start()
time.sleep(2)

# 拍照测试
r = picam2.capture_request()
r.save("main", "test.jpg")
r.release()
print("Capture OK, file size:", __import__('os').path.getsize("test.jpg"))

picam2.stop()
picam2.close()
EOF
```

## CI 集成（可选）

```yaml
# .github/workflows/test.yml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
      - run: uv python install 3.13
      - run: uv sync
      - run: uv run pytest tests/
```

注意：CI 环境中 Picamera2 不可用，所有测试必须通过 mock 实现，或在 `pyproject.toml` 中将 picamera2 标记为可选依赖。
